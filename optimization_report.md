# 周末规划助手 Demo 优化记录

本文档根据当前 `demo.html` 状态整理，记录已修复问题、当前策略和后续优化点。

## 1. 当前状态概览

Demo 已从早期“生成活动 + 餐厅方案”的原型，升级为具备以下能力的 Agent 化规划助手：

- 支持自然语言意图解析和多轮上下文继承。
- 支持真实 POI 搜索、天气查询、路线距离估算和计划卡生成。
- 支持亲子、朋友、情侣、宠物、两天一夜等场景。
- 支持“换餐厅”“换活动”“换近一点”“预算低一些”等连续调整。
- 用户视角已清理掉内部指标、技术说明、冗余反馈按钮。
- 技术视角保留 ReAct/工具调用过程，便于演示 Agent 机制。

## 2. 已完成优化

### 2.1 方案完整性

| 问题 | 当前修复 |
|---|---|
| 全天方案缺少午餐/晚餐 | 全天计划拆分午餐和晚餐 |
| 人群标签显示异常 | 修复亲子标签，显示如“亲子·5岁儿童·3人” |
| 费用显示“待确认” | 改为人均预估，综合活动、餐饮、住宿等费用 |
| 当前时间后规划 | 默认开始时间按当前时间向上取整，不再回到过去时间 |
| 工作日误规划今天夜间 | 工作日的周末/亲子/放电语境默认规划最近周六下午 |

### 2.2 天气和日期

| 问题 | 当前修复 |
|---|---|
| 天气与方案日期不一致 | `fetchWeather` 跟随 `intent.targetDate` 查询 |
| 周末方案使用当前实时天气 | 周末方案查周六/周日预报 |
| 计划标题误显示“今晚方案” | 有目标日期时显示如“周六下午方案” |

### 2.3 POI 推荐质量

| 问题 | 当前修复 |
|---|---|
| 医院被推荐为亲子活动 | 增加无效活动 POI 黑名单过滤 |
| 亲子放电场景不精准 | 强化儿童乐园、蹦床、运动馆、科技馆等关键词 |
| 故宫夜间/临闭馆被推荐 | 增加故宫预约、闭馆、营业时间、停止入院规则 |
| 营业时间不准 | 尝试通过高德 POI Detail 获取真实详情，并用规则兜底 |
| 远距离 POI 被高评分顶上来 | 半日计划距离优先；近距离模式下按距离优先排序 |

### 2.4 多轮调整

| 用户操作 | 当前行为 |
|---|---|
| 预算低一些 | 继承原人群、时间、孩子年龄和偏好，优先低价方案 |
| 换个近一点的 | 根据当前距离收紧到 5km / 3km / 2km，并按距离优先排序 |
| 不想吃这家，换个餐厅 | 局部重绘，只替换餐厅，不重新完整思考 |
| 换套方案 | 在完整候选中切换备选方案 |
| 突发带狗 | 重新规划为宠物友好场景 |

### 2.5 用户界面

| 问题 | 当前修复 |
|---|---|
| 用户视角展示过多内部说明 | 删除指标、风险控制、工具可靠性等技术面板 |
| 欢迎区权限说明过重 | 简化为用户可理解的助手说明 |
| 计划卡有冗余提示 | 删除“推荐不代表最终决定”等重复说明 |
| 反馈按钮和继续问重复 | 删除上方反馈按钮，只保留“继续问”操作入口 |
| App 标识拥挤 | 调整为 iOS 毛玻璃卡片，并将标题改为“周末规划助手” |

## 3. 关键策略说明

### 3.1 上下文继承

调整类输入会继承上一轮方案：

- 人群类型。
- 人数。
- 儿童年龄。
- 出发时间。
- 目标日期。
- 天气目标。
- 交通方式和强度。

避免出现“亲子方案调整预算后变成朋友局”的问题。

### 3.2 局部重绘

当前已实现餐厅局部重绘：

```text
用户：不想吃这家，帮我换个餐厅
系统：保留活动和时间，只替换餐厅节点
```

计划卡会用：

- 活动节点：锁定。
- 餐厅节点：已替换。
- 标题：局部重绘：已更换餐厅。

### 3.3 近距离策略

“换个近一点的”不再只是文案，而是明确改变距离阈值：

| 当前活动距离 | 新阈值 |
|---|---:|
| > 8km | 5km |
| 3–8km | 3km |
| ≤ 3km | 2km |

## 4. 历史审查问题状态

| 原问题 | 当前状态 |
|---|---|
| 用户输入 XSS 风险 | 当前用户输入已通过 `escapeHtml` 输出 |
| `showAlternatives()` 空壳 | 已实现换套方案逻辑 |
| 预算字段缺失 | JS 意图解析已支持预算和低价诉求 |
| 多轮状态缺失 | 已通过 `globalCurrentPlan` 继承上下文 |
| 餐厅不满意后重跑完整流程 | 已改为局部重绘餐厅 |
| 距离约束不明确 | 已加入近距离阈值收紧规则 |
| 天气和行程日期不一致 | 已用 `targetDate` 对齐 |
| 无效活动 POI | 已增加黑名单过滤 |

仍需关注：

- `agent.py` 和 `mock_api.py` 是 Python 原型，未完全同步 `demo.html` 的全部最新策略。
- 高德 POI 详情营业时间字段不稳定，需要进一步验证字段来源。
- onclick 字符串拼接仍有可维护性风险，后续可改为事件委托和 dataset。

## 6. 验收用例

| 用例 | 预期 |
|---|---|
| 工作日输入“今天下午带 5 岁孩子放电” | 生成最近周六下午亲子方案 |
| 亲子方案后点“预算低一些” | 仍保持亲子，不变朋友局 |
| 方案距离 16km 后点“换个近一点的” | 新方案优先控制在 5km 内 |
| 输入“不想吃这家，帮我换个餐厅” | 不重新思考，只替换餐厅 |
| 输入“故宫”且时间较晚 | 不推荐临闭馆/需预约的故宫临时行程 |
| 高德返回医院类 POI | 不作为活动推荐 |
